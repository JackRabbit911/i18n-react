# i18n-react
## Модуль интернационализации для React TS
- Не требует размещения на клиенте всей таблицы переводов
- Получает необходимые переводы по запросу с сервера в виде объекта ключ - значение
- Кэширует полученные данные. Кэшированные данные повторно не запрашиваются
- Есть возможность установки лимита кэша
- Есть возможность получения всей карты переводов за один запрос
- Вы можете реализовать любой контракт с бэкендом
- Не зависит ни от каких библиотек, кроме самого React
## Использование
*Для использования Вам нужна лишь папка i18n, остальное для демонстрации*
Оберните Ваши компоненты.
```typescript
const location = useLocation()

return (
    <TranslateProvider deps={[location.pathname]}>
        ...
    </TranslateProvider>
)
```
*`useLocation()` использовать не обязательно. Вы можете указать любой массив зависимостей аналогично `useEffect(()=>{...}, [...])`. Важно! Завасимость должна быть реактивной на уровне `<TranslateProvider>`*

Кде-то в компоненте:
```typescript
const __ = useTranslate() //Это реактивная функция перевода. Назовите её как угодно

return (
    <>
        <h1>{__('Title')}
        <Child __={__} /> {/*Выглядит смешно*/}
    </>
)
```
Child.tsx
```typescript
type Props = {
    __: GetText; //Функция уже типизирована
}

const Child = ({ __ }: Props) => {
    return (
        <div>
            {__('Up to % symbols', 8)} {/*Не более 8 символов*/}
        </div>
    )
}
```
*Вы можете неоднократно вызывать хук `useTranslate()` на любом уровне вложенности внутри `<TranslateProvider>` или передавать функцию перевода пропсами*

Функция запроса к серверу должна быть типизирована вот так:
```typescript
export type TranslateType = {
  [key: string]: string;
}
export type FetchTranslateType = (lang: string, keys: string[]) => Promise<TranslateType>
//или так: (если вы хотите получать всю таблицу переводов одним запросом)
export type FetchTranslateType = (lang: string, keys: null) => Promise<TranslateType>
```
*В случае использования `fetchAllMap()` установите параметр delay в 0*

Рекомендация: Установите ДЕМО и посмотрите, где работает правильно, а где нет. Это очень просто:
```bash
git clone https://github.com/JackRabbit911/i18n-react
cd i18n-react
npm i
npm run serve
```
*В косоли Вы увидите на каком порту поднялся сервер. В браузере перейдите по адресу `localhost:PORT`*

Посмотрите пример конфигурации (файл i18n/config.ts) и пример функций fetchTranslate() и fetchAllMap() (файл i18n/utils.ts)

На этом всё. Удачи!
